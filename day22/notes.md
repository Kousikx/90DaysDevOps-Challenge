## git add vs git commit

git add moves changes to the staging area.
git commit saves those staged changes permanently in history.

Add prepares.
Commit records.

---

## What is the staging area?

The staging area lets me control what goes into the next commit.
It prevents accidental commits.

---

## What does git log show?

- Commit hash
- Author
- Date
- Commit message

It shows project history.

---

## What is .git folder?

It stores:
- All commits
- Branch info
- Repository metadata

If deleted, the project loses version history.

---

## Working Directory vs Staging Area vs Repository

Working Directory:
Where I edit files.

Staging Area:
Where I prepare files.

Repository:
Where commits are permanently saved.
